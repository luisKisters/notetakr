PeopleSource (many; device- or server-side) feeds the picker & attendee auto-match. CRMSink (one adapter per CRM; server-side TypeScript in Convex) receives meeting notes. Apple Contacts is a source, never a sink. Nothing is merged anywhere — only mirrored or pushed, so three places stay tolerable.

Pipeline — meeting end to CRM note

  1. macMeeting ends — note + transcript finalized locally (existing behavior, unchanged).
  2. macPush — outbox uploads meeting, note, transcript segments to Convex. Idempotent upsert keyed by local record ID; retry-safe; skipped when localOnly.
  3. convexSummarize — action calls OpenRouter (Kimi K2.7) with the transcript, writes summary onto the meeting. UI shows waiting → generating → done.
  4. convexMatch — participant emails resolved against people (Twenty mirror). Unmatched names recorded on the meeting for a later “link / create in CRM” affordance — they never block the push.
  5. convexPush to Twenty — scheduler fires upsertMeetingNote(): one note (summary on top, transcript beneath), attached to all matched persons. crmNoteId stored → retries update, never duplicate.
  6. macStatus surfaces — per-meeting pushStatus: pending | pushed | failed | skipped shown as a quiet indicator in the note’s property panel.

Risks & open items

  • Attendee emails are best-effort — some calendar accounts expose names only; those participants won’t auto-match (accepted as-is; quiet banner above the footer when a CRM is connected).
  • LLM cost + privacy asterisk — server summaries mean “transcripts are processed in the cloud when sync is on”; needs one honest line in the README/settings. Signed-out users keep the existing local OpenRouter path.
  • Apple Developer membership ($99/y) — not needed for any of P1–P4 (Google-only auth), but unlocks notarization + Sign in with Apple later.
  • Kimi K2.7 model slug — confirm the exact OpenRouter slug at implementation time; it lives in a Convex env var, not code.
